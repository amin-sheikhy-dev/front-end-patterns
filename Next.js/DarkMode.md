تنظیم darkMode: 'class' به Tailwind میگه که به جای استفاده از تنظیمات پیش‌فرض مرورگر (سیستم عامل)، باید به صورت دستی با اضافه کردن کلاس dark به یک تگ والد (معمولاً html یا body) حالت تاریک رو فعال کنی

```js
/** @type {import('tailwindcss').Config} */
export default {
  darkMode: 'class', // برای دارک مود ضروریه
  content: ['./src/**/*.{html,js,jsx,ts,tsx}', './public/index.html'],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

برای تغییر رنگ بادی هنگام اعمال دارک مود ضروریه

مشخص میکنیم که در حالت عادی پس‌زمینه سفید و در حالت دارک مود، پس‌زمینه خاکستری تیره (gray-900) باشه.

این کد باید در فایل اصلی CSS شما (مثلاً index.css یا globals.css) قرار بگیره.

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  body {
    @apply bg-white dark:bg-gray-900;
  }
}
```

این کد مربوط به تنظیمات `next-themes` برای `Dark Mode` در `Next.js` هست:

و `attribute='class'` برای اعمال تم کلاس `dark` یا `light` به المنت `<html>` اضافه میشود

و `defaultTheme='system'` `'dark'` `'light'` تم اولیه سیستم کاربر

و `enableSystem={false}` اجازه میدهد از تم سیستم به عنوان `fallback` استفاده شود

و `disableTransitionOnChange` ترنزیشن های `CSS` را در زمان تغییر تم غیرفعال میکند

```tsx
'use client';

import { ThemeProvider as NextThemesProvider } from 'next-themes';
import React from 'react';

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  return (
    <NextThemesProvider attribute="class" defaultTheme="dark" enableSystem={false} disableTransitionOnChange>
      {children}
    </NextThemesProvider>
  );
}
```

این کد مربوط به کامپوننت `ToggleTheme` برای تغییر تم با استفاده از `next-themes` هست:

و `mounted:` استفاده از `mounted` برای جلوگیری از خطای Hydration در `Next.js`. چون `next-themes` در ابتدا تم رو تشخیص نمیده، `mounted` تا زمانی که کامپوننت به Client برسه مقدار `false` هست و `null` برمیگردونه.

و `useTheme:` یک هوک از `next-themes` هست که `theme` (تم فعلی) و `setTheme` (تابع تغییر تم) رو برمیگردونه.

```tsx
'use client';

import { useTheme } from 'next-themes';
import { useEffect, useState } from 'react';

export default function ToggleTheme() {
  const [mounted, setMounted] = useState(false);
  const { theme, setTheme } = useTheme();

  // eslint-disable-next-line
  useEffect(() => setMounted(true), []);

  if (!mounted) return null;

  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')} className="">
      {theme === 'dark' ? '🌙' : '☀️'}
    </button>
  );
}
```

این کد مربوط به `layout.tsx` اصلی در `Next.js` با پشتیبانی از `Dark Mode` با `next-themes` هست:

و `suppressHydrationWarning:` کامنت خط `12` اشاره میکنه که این ویژگی روی تگ `html` ضروری است تا از خطاهای هیدریشن `(Hydration)` جلوگیری بشه. دلیلش اینه که کلاس تم `(dark یا light)` ممکنه در سمت سرور و کلاینت متفاوت باشه و این هشدار رو خاموش میکنه.

و `ThemeProvider:` کامپوننت `ThemeProvider` که قبلاً ساختیم رو ایمپورت و استفاده میکنیم تا تمام فرزندان به تم دسترسی داشته باشن.

```tsx
import type { Metadata } from 'next';
import './globals.css';
import { ThemeProvider } from '@/components/theme-provider';

export const metadata: Metadata = {
  title: 'Create Next App',
  description: 'Generated by create next app',
};

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    // روی تگ html برای جلوگیری از خطای هیدریشن به دلیل کلاس تم ضروری است suppressHydrationWarning
    <html lang="en" suppressHydrationWarning>
      <body>
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  );
}
```
